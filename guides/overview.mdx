## Overview

ShopifyMax powers high-volume stores with checkout flows that complete in under three seconds. The platform manages catalogs exceeding 100000 products while maintaining reliable performance. You configure cart and payment variables directly in your integration code to keep transactions secure and traceable.

## Checkout Speed Optimization

You enable edge caching and pre-rendered payment forms to hit sub-three-second targets. ShopifyMax automatically compresses assets and prioritizes critical checkout scripts.

<Callout kind="tip">
  Test your checkout page with real network conditions before launch.
</Callout>

## Catalog Management

You organize products in hierarchical collections and apply bulk updates through the dashboard. The system indexes every item for instant search across storefronts.

## Cart Token Workflows

You pass the `CART_TOKEN` variable at session start to track abandoned carts and restore sessions across devices.

<Tabs>
  <Tab title="JavaScript" icon="code">
    Store the token in local storage for later retrieval.
  </Tab>
  <Tab title="Python" icon="terminal">
    Attach the token to every subsequent API call.
  </Tab>
</Tabs>

## Payment Key Handling

You supply the `PAYMENT_KEY` variable when initializing the payment gateway. ShopifyMax validates the key on each transaction to prevent unauthorized charges.

## Examples

<CodeGroup tabs="JavaScript,Python">
```javascript
const cartToken = "your_cart_token_here";
const paymentKey = "your_payment_key_here";
fetch("https://api.example.com/checkout", {
  headers: { "X-Cart-Token": cartToken, "X-Payment-Key": paymentKey }
});
```
```python
cart_token = "your_cart_token_here"
payment_key = "your_payment_key_here"
requests.post("https://api.example.com/checkout", headers={
    "X-Cart-Token": cart_token,
    "X-Payment-Key": payment_key
})
```
</CodeGroup>

## Best Practices

<Expandable title="Secure your keys" default-open="false">
  Never expose `PAYMENT_KEY` in client-side bundles. Rotate keys quarterly through the dashboard.
</Expandable>

Store `CART_TOKEN` values server-side when possible. Monitor checkout latency in your analytics dashboard and adjust caching rules when average times exceed two seconds.
