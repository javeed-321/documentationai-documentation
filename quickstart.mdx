## Prerequisites

<Callout kind="info">
You need a Git provider account (GitHub, GitLab, or Bitbucket) and a Vercel account. Create a repository with a supported framework such as Next.js, React, or Vue.
</Callout>

## Connect Your Repository

<Steps>
<Step title="Import the project" icon="download">
Open the Vercel dashboard and select Import Git Repository. Choose your repository from the list.
</Step>
<Step title="Configure settings" icon="settings">
Vercel automatically detects the framework and sets the build command, output directory, and install command.
</Step>
<Step title="Deploy" icon="rocket">
Click Deploy. Vercel builds the project and creates a unique preview URL.
</Step>
</Steps>

## Preview Deployments

Every push to a non-production branch generates a unique preview deployment. Share the preview URL with your team for instant feedback before merging.

## Production Rollout

Merge to the main branch to trigger an automatic production deployment. Vercel promotes the build to your live domain with zero downtime.

## Deploy with the CLI

Use the Vercel CLI for projects without Git integration.

<CodeGroup tabs="npm,yarn">
```bash
npm i -g vercel
vercel
```
```bash
yarn global add vercel
vercel
```
</CodeGroup>
