## Overview

ShopifyMax returns structured error responses for checkout operations. Every error includes a `code` field and a `message` that references the affected `{CART_TOKEN}` or `{PAYMENT_KEY}`.

<Callout kind="info">
Include the exact `{CART_TOKEN}` value from the failing request when contacting support.
</Callout>

## Checkout Decline Codes

<Expandable title="Error: CHECKOUT_DECLINED" default-open="false">

**Problem**

```json
{
  "error": "CHECKOUT_DECLINED",
  "message": "Checkout declined for cart token cart_9k2pL4mX"
}
```

**Cause**

The payment processor rejected the transaction after the cart contained more than 100000 products or the total exceeded the merchant limit.

**Fix**

Split the cart into smaller batches under 50000 products each and retry with the same `{CART_TOKEN}`.

```bash
curl -X POST https://api.example.com/v1/checkout \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -d '{"cart_token":"cart_9k2pL4mX","split":true}'
```

</Expandable>

<Expandable title="Error: CHECKOUT_TIMEOUT" default-open="false">

**Problem**

```json
{
  "error": "CHECKOUT_TIMEOUT",
  "message": "Checkout exceeded 3s limit"
}
```

**Cause**

Network latency or an oversized payload prevented completion within the required window.

**Fix**

Reduce payload size and ensure the request originates from a region with sub-100 ms latency to `api.example.com`.

</Expandable>

## CART_TOKEN Errors

<Expandable title="Error: INVALID_CART_TOKEN" default-open="false">

**Problem**

```json
{
  "error": "INVALID_CART_TOKEN",
  "message": "cart_9k2pL4mX is expired or malformed"
}
```

**Cause**

The token was generated more than 30 minutes ago or contains invalid characters.

**Fix**

Generate a fresh token and re-add items before attempting checkout again.

```javascript
const token = await shopifyMax.carts.create();
```

</Expandable>

## PAYMENT_KEY Failures

<Expandable title="Error: INVALID_PAYMENT_KEY" default-open="false">

**Problem**

```json
{
  "error": "INVALID_PAYMENT_KEY",
  "message": "Payment key pk_live_abc123 rejected"
}
```

**Cause**

The `{PAYMENT_KEY}` is either revoked, scoped to the wrong merchant, or used after its 24-hour rotation window.

**Fix**

Rotate the key in the dashboard and update the environment variable before the next request.

</Expandable>

## Rate Limit Handling

<Expandable title="Error: RATE_LIMIT_EXCEEDED" default-open="false">

**Problem**

HTTP 429 response with header `Retry-After: 60`.

**Cause**

More than 1000 requests per minute were sent using the same `{CART_TOKEN}`.

**Fix**

Implement exponential backoff starting at 1 second and respect the `Retry-After` value.

```python
import time
time.sleep(int(response.headers["Retry-After"]))
```

</Expandable>
