## Common API Errors

Vercel returns standard HTTP status codes with JSON error bodies. Each error includes a `code` and `message` field. Review the examples below to diagnose issues during API calls and deployments.

<Expandable title="429 Too Many Requests" default-open="false">
**Problem**

```json
{
  "error": {
    "code": "rate_limit_exceeded",
    "message": "Rate limit exceeded. Try again in 45 seconds."
  }
}
```

**Cause**

The API token exceeded 100 requests per minute.

**Fix**

Add exponential backoff to your client. Example:

```javascript
async function fetchWithRetry(url, options, retries = 3) {
  const res = await fetch(url, options);
  if (res.status === 429 && retries > 0) {
    const wait = Math.pow(2, 3 - retries) * 1000;
    await new Promise(r => setTimeout(r, wait));
    return fetchWithRetry(url, options, retries - 1);
  }
  return res;
}
```
</Expandable>

<Expandable title="Build Failed" default-open="false">
**Problem**

```json
{
  "error": {
    "code": "build_failed",
    "message": "Command \"npm run build\" exited with 1"
  }
}
```

**Cause**

A build step failed due to a missing dependency or syntax error in source code.

**Fix**

Run the build locally with `vercel build` to reproduce, then inspect the `vercel-output` directory for logs. Fix the failing command and redeploy.
</Expandable>

<Expandable title="Invalid Configuration" default-open="false">
**Problem**

```json
{
  "error": {
    "code": "invalid_config",
    "message": "vercel.json: unknown property \"buildCommand\""
  }
}
```

**Cause**

The `vercel.json` file contains an unsupported key or incorrect value type.

**Fix**

Validate your configuration against the current schema. Replace the invalid key:

```json
{
  "buildCommand": "npm run build"
}
```
</Expandable>

<Expandable title="Permission Denied" default-open="false">
**Problem**

```json
{
  "error": {
    "code": "forbidden",
    "message": "Insufficient permissions to access project"
  }
}
```

**Cause**

The API token lacks the required scope for the requested project or team.

**Fix**

Regenerate the token at `https://vercel.com/account/tokens` with at least `Project:Read` and `Deployment:Write` scopes, then update your `Authorization` header.
</Expandable>

<Callout kind="info">
Always include the request ID from response headers when contacting support.
</Callout>
