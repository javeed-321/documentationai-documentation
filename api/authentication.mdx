## Overview

All requests to the ShopifyMax API require authentication. You authenticate using either an API key passed in a header or session tokens for cart and payment operations. Generate keys from the dashboard at `https://dashboard.example.com`.

## Authentication Methods

<Tabs>
  <Tab title="API Key" icon="key">
    Include the API key in the `Authorization` header for every request.

    <ParamField header="Authorization" param-type="string" required="true">
      Use the format `Bearer YOUR_API_KEY`.
    </ParamField>
  </Tab>
  <Tab title="Session Tokens" icon="lock">
    Pass `CART_TOKEN` and `PAYMENT_KEY` as query parameters or in request bodies for checkout flows.

    <ParamField query="cart_token" param-type="string" required="false">
      Token generated for the current cart session.
    </ParamField>

    <ParamField body="payment_key" param-type="string" required="false">
      Key required to process payments under 3 seconds.
    </ParamField>
  </Tab>
</Tabs>

## Implementation Examples

<CodeGroup tabs="JavaScript,Python,cURL">
```javascript
const response = await fetch('https://api.example.com/v1/products', {
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  }
});
```

```python
import requests

headers = {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
}
response = requests.get('https://api.example.com/v1/products', headers=headers)
```

```bash
curl -H "Authorization: Bearer YOUR_API_KEY" \
     https://api.example.com/v1/products
```
</CodeGroup>

## Security Considerations

<Callout kind="alert">
Store API keys and tokens securely. Never expose `CART_TOKEN` or `PAYMENT_KEY` in client-side code or logs. Rotate keys regularly from the dashboard.
</Callout>
