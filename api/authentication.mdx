## Overview

All requests to the ShopifyMax API require authentication. You pass credentials as HTTP headers. The platform accepts two variables: `PAYMENT_KEY` for payment operations and `CART_TOKEN` for cart sessions. Requests without valid credentials return HTTP 401.

Use `https://api.example.com` as the base URL for all authenticated endpoints.

## Authentication methods

<Tabs>
  <Tab title="PAYMENT_KEY" icon="key">
    Send the `PAYMENT_KEY` header for payment and order endpoints. This key identifies your merchant account.

    <ParamField header="PAYMENT_KEY" param-type="string" required="true">
      Your merchant payment credential. Obtain it from the dashboard at `https://dashboard.example.com/keys`.
    </ParamField>
  </Tab>
  <Tab title="CART_TOKEN" icon="lock">
    Send the `CART_TOKEN` header when modifying carts or checking out. Tokens expire after 30 minutes of inactivity.

    <ParamField header="CART_TOKEN" param-type="string" required="true">
      Session token returned by the `/cart` endpoint. Use it to associate subsequent requests with an active cart.
    </ParamField>
  </Tab>
</Tabs>

## Implementation examples

<CodeGroup tabs="cURL,JavaScript,Python">
```bash
curl -X POST https://api.example.com/checkout \
  -H "PAYMENT_KEY: pm_live_1234567890" \
  -H "CART_TOKEN: ct_abcdef123456" \
  -H "Content-Type: application/json" \
  -d '{"items":[{"sku":"ABC","qty":1}]}'
```

```javascript
const response = await fetch("https://api.example.com/checkout", {
  method: "POST",
  headers: {
    "PAYMENT_KEY": "pm_live_1234567890",
    "CART_TOKEN": "ct_abcdef123456",
    "Content-Type": "application/json",
  },
  body: JSON.stringify({ items: [{ sku: "ABC", qty: 1 }] }),
});
```

```python
import requests

headers = {
    "PAYMENT_KEY": "pm_live_1234567890",
    "CART_TOKEN": "ct_abcdef123456",
    "Content-Type": "application/json",
}
data = {"items": [{"sku": "ABC", "qty": 1}]}
response = requests.post("https://api.example.com/checkout", json=data, headers=headers)
```
</CodeGroup>

## Security recommendations

<Callout kind="alert">
Never expose `PAYMENT_KEY` or `CART_TOKEN` in client-side code or public repositories. Rotate keys immediately if you suspect leakage. Always use HTTPS.
</Callout>
